# Excel-анализатор для dogovor

Веб-приложение для анализа структуры Excel-файлов с договорами. Автоматически определяет колонки, типы данных, сопоставляет с полями системы **dogovor** и выдаёт рекомендации.

## Возможности

- Загрузка файла drag-and-drop (.xlsx, .xls, .csv)
- Автоопределение строки заголовков (шапки) — если не уверен, предлагает выбрать вручную
- Анализ каждой колонки: тип данных, заполненность, уникальность, примеры значений
- Автоматическое сопоставление колонок с полями модели Contract (номер, контрагент, сумма, статус, даты...)
- Выявление отсутствующих обязательных полей
- Формирование текстовых рекомендаций для адаптации файла под dogovor
- Скачивание отчёта в JSON

## Как это работает

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│ Загрузка     │────→│ Анализ шапки     │────→│ Чтение таблицы  │
│ Excel/CSV    │     │ (поиск строки с  │     │ от найденной    │
│              │     │  заголовками)    │     │ строки          │
└─────────────┘     └──────────────────┘     └─────────────────┘
                                                       │
                                                       ↓
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│ Отчёт       │←────│ Рекомендации     │←────│ Анализ колонок  │
│ (HTML+JSON) │     │ + сопоставление  │     │ (типы, пустоты, │
│             │     │ с dogovor        │     │  примеры)       │
└─────────────┘     └──────────────────┘     └─────────────────┘
```

### Этап 1: Определение шапки

Файл читается без заголовков. Первые 15 строк сканируются на совпадение со словарём полей dogovor. Каждая строка получает рейтинг:

- +10 за точное совпадение с ключевым словом (например, «Номер» → `number`)
- +3 за частичное совпадение
- +1 за текст без цифр (признак заголовка)
- +5 за покрытие колонок (если строка заполнена не менее чем на 40%)

Если лучший кандидат набирает ≥15 баллов и вдвое больше второго — уверенность `high`, анализ выполняется автоматически. Иначе пользователю показываются 5 лучших кандидатов для выбора.

### Этап 2: Анализ колонок

Для каждой колонки определяется:
- Тип данных (текст, число, дата)
- Количество непустых значений
- Процент пропусков
- Количество уникальных значений
- Первые 5 значений (примеры)

### Этап 3: Сопоставление с dogovor

Поддерживаемые поля (словарь в `app.py`):

| Поле dogovor | Ожидаемые названия колонок |
|-------------|---------------------------|
| `number` | Номер, №, регистрационный |
| `name` | Наименование, название, договор |
| `counterparty` | Контрагент, партнёр, организация |
| `subject` | Предмет, содержание, описание |
| `amount` | Сумма, цена, стоимость |
| `status` | Статус, этап, состояние |
| `responsible` | Ответственный, исполнитель, менеджер |
| `notes` | Примечание, комментарий, заметки |
| `contract_type` | Тип, вид |
| `received_date` | Дата получения, дата |
| `signing_date` | Дата подписания |

### Этап 4: Рекомендации

Формируется текстовый отчёт с:
- Итоговой статистикой (строк, колонок, номер шапки)
- Соответствием полей
- Списком отсутствующих обязательных полей (number, name, counterparty)
- Предложениями для доработки программы dogovor под загруженный файл

## Установка и запуск

```bash
docker compose up -d
# или вручную:
pip install -r requirements.txt
python app.py
```

Приложение будет доступно на порту 5100.

## API endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Веб-интерфейс |
| POST | `/api/analyze` | Загрузить и проанализировать файл |
| GET | `/api/download/<filename>` | Скачать сохранённый отчёт |

### Параметры `/api/analyze`

- `file` — файл .xlsx/.xls/.csv (multipart/form-data)
- `header_row` (опционально) — номер строки заголовков (1-indexed)

Если `header_row` не указан, анализатор пытается определить шапку автоматически. При низкой уверенности возвращает `status: "needs_header_selection"` с кандидатами.
